# E-commerce Analytics com SQL

Projeto acadêmico desenvolvido com foco em modelagem relacional, normalização de dados e consultas SQL para análise operacional de um e-commerce.

## Objetivo

Desenvolver um banco de dados relacional para gestão de pedidos de um e-commerce, aplicando boas práticas de modelagem e consultas analíticas para apoiar decisões de negócio.

## Tecnologias utilizadas

- SQL Server
- SQL
- Modelagem Relacional
- Draw.io
- Git
- GitHub

```txt
scripts/
images/

```

## Modelagem de dados

O banco foi estruturado com entidades relacionadas ao processo de vendas:

- Clientes
- Endereços
- Categorias
- Produtos
- Pedidos
- Itens de pedido
- Pagamentos

## Normalização aplicada

✅ Primeira Forma Normal (1FN)  
✅ Segunda Forma Normal (2FN)  
✅ Terceira Forma Normal (3FN)  

## Funcionalidades implementadas

✅ Criação de tabelas  
✅ Chaves primárias e estrangeiras  
✅ Integridade referencial  
✅ Relacionamentos entre entidades  
✅ Consultas analíticas  

## Análises realizadas

- Top clientes por volume de compras
- Produtos mais vendidos
- Pedidos pendentes
- Pagamentos aprovados e pendentes
- Clientes sem pedidos
- Priorização de pedidos
- Total de pedidos por cliente

## Insights de negócio

Com base nas análises realizadas, foram identificadas oportunidades como:

- Reforço de estoque de produtos com maior demanda
- Monitoramento de pagamentos pendentes
- Identificação de clientes com maior ticket médio
- Produtos com baixo desempenho comercial

## Aprendizados

Neste projeto foram aplicados conceitos de modelagem de dados, normalização, integridade referencial, consultas analíticas e geração de insights para tomada de decisão.
